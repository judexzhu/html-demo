ARO HCP via ASO — Deployment Architecture
Deploying ARO HCP with Swift using Azure Service Operator (ASO) from a local Kind cluster.
⚠ASO-based deployment. Azure Service Operator runs in a local Kind cluster alongside CAPI/CAPZ controllers. It manages all Azure resources (VNet, Key Vault, Managed Identities, Role Assignments) as Kubernetes CRDs, then creates the ARO HCP cluster resource declaratively. Requires networkType: Other with manual Cilium CNI installation.
